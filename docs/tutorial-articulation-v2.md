# VimSkillHub 试用教程：上传并使用 ArticulationV2 Skill

> 面向初学者的完整操作指南，以 `ArticulationV2.skill` 为例。

---

## 前置准备

1. 使用公司邮箱（`@eazillion.com` 或 `@vim-technology.com`）注册账号
2. 准备好你要上传的 `.skill` 文件（本教程使用 `ArticulationV2.skill`）

---

## 第一步：注册账号

1. 打开 VimSkillHub 网站
2. 点击右上角 **Register**
3. 填写：
   - **Email**: 你的公司邮箱，例如 `zhangsan@vim-technology.com`
   - **Password**: 设置密码（至少 6 位）
   - **Display Name**: 你的显示名称，例如 `张三`
4. 点击 **Create Account**

> 注意：只有 `@eazillion.com` 和 `@vim-technology.com` 邮箱可以注册。

---

## 第二步：上传 Skill

1. 点击顶部导航栏的 **Upload**（上传）
2. 填写 Skill 信息：

| 字段 | 填写示例 | 说明 |
|------|----------|------|
| **Title** | `ArticulationV2 - Dan Koe 写作系统` | Skill 的名称 |
| **Short Description** | `基于 Dan Koe 显性教学 + 隐性技巧的说服性写作系统。按受众类型和认知操作自动选择写作策略。` | 一句话概括 |
| **Documentation** | 见下方 | 详细说明（支持 Markdown） |
| **Category** | `Workflow` | 从下拉菜单选择最合适的分类 |
| **Tags** | `writing, persuasion, dan-koe, framework` | 用英文逗号分隔的标签 |
| **Version** | `2.0.0` | 版本号 |
| **GitHub URL** | （可选）如果有 GitHub 仓库可以填 | 关联代码仓库 |
| **File Upload** | 选择 `ArticulationV2.skill` 文件 | 点击上传按钮选择文件 |

### Documentation 建议内容（粘贴到文本框）

```markdown
# ArticulationV2 - Dan Koe 写作系统

一个从 Dan Koe 的显性教学和隐性写作模式中提炼的说服性写作系统。

## 核心能力

- **受众识别**：4 种受众类型（学习者/同行/上级/专家）
- **认知操作诊断**：自动识别你要对读者做什么（认知翻转/更新/深化/构建）
- **技巧选择矩阵**：根据受众 × 操作自动推荐最佳写作技巧
- **23+ Definition Flips 示例库**
- **11 种显性技巧 + 8 种隐性技巧**

## 使用场景

- 撰写文章、Newsletter、演讲稿、内部 Memo
- 说服性写作：重新定义概念、建立框架、引导读者得出结论
- 任何需要"让人听进去"的沟通

## 三阶段工作流

1. Phase 1：确认受众类型
2. Phase 2：确认认知操作（翻转/更新/深化/构建）
3. Phase 3：自动选择技巧并执行写作

## 关键原则

- Definition Flip > 论证（重新定义比争论更有力）
- 阶梯式 > 并列式（每个要点建立在前一个之上）
- 教育即说服（让读者自己得出你的结论）
```

3. 点击 **Publish Skill** 发布

---

## 第三步：浏览你发布的 Skill

1. 发布成功后会自动跳转到 Skill 详情页
2. 你可以看到：
   - 标题、描述、文档（Markdown 渲染后的效果）
   - 下载按钮、Star 按钮
   - 评分区域、评论区域
   - 右侧边栏的统计信息（Stars、Downloads、Rating）

---

## 第四步：下载 Skill

1. 进入 **Explore** 页面，搜索 `Articulation`
2. 点击 Skill 卡片进入详情页
3. 点击绿色 **Download** 按钮
4. 文件会自动下载到本地
5. 下载次数会自动 +1

---

## 第五步：给 Skill 打 Star

1. 在 Skill 详情页，点击 **Star** 按钮
2. 按钮变为蓝色高亮 = 已 Star
3. 再次点击可以取消 Star

---

## 第六步：评分

1. 在详情页的 **Rating** 区域
2. 看到 5 颗星，鼠标悬停预览
3. 点击你想打的星数（1-5 星）
4. 评分会立即生效，平均分会重新计算

---

## 第七步：评论

1. 滚动到详情页底部的 **Comments** 区域
2. 在输入框中写下你的评价，例如：
   > "Definition Flip 的思路很有启发，已经在上周的 Newsletter 里试过了，转化率提升明显。"
3. 点击 **Send** 发送
4. 你的评论会立即显示，其他同事也能看到

---

## 第八步：查看个人主页

1. 点击右上角你的用户名
2. 进入 Profile 页面
3. 可以看到：
   - 你发布的所有 Skills
   - 获得的总 Stars、总 Downloads
   - 编辑个人简介和头像

## 关于 ArticulationV2 这个 Skill 本身

这是一个**说服性写作系统**，核心流程是：

```
确认受众 → 诊断认知操作 → 自动选择技巧 → 执行写作
```

### 4 种受众类型

| 类型 | 描述 | 写作策略 |
|------|------|----------|
| A1 学习者 | 知道基础，没你深 | 慢慢引导，给 "aha moment" |
| A2 同行 | 水平相当，领域不同 | 快速给新输入，他们自己连点 |
| A3 上级 | 比你懂得多，需要你的专业 | 高密度，跳过铺垫，直接给洞察 |
| A4 专家 | 需要一步步指导 | 先给认知地图，再给小步骤 |

### 4 种认知操作

| 操作 | 含义 | 核心技巧 |
|------|------|----------|
| Cognition Shift | 翻转他们的认知 | Definition Flip |
| Cognition Update | 填补认知空白 | Cross-Domain Synthesis |
| Cognition Deepening | 让已知观点更深入 | Evolution Ladder |
| Cognition Construction | 从零建立理解 | Teach-your-past-self |

### Definition Flip 示例

| 原始定义 | Dan 的重新定义 |
|----------|---------------|
| 工作生活平衡 = 时间平均分配 | 平衡 = 平庸的中间地带 → 应该追求"对比"（深度工作 + 深度休息交替） |
| 天才 = 极高智商 | 天才 = "继续思考的能力" → 不是够不够聪明，是有没有太早停下 |
| 利基市场 = 专注一个话题 | 你的利基 = 你的使命 → 服务使命的一切都是你的品牌 |
| 写作 = 把字写在纸上 | 写作 = 思考的可见形态 → 写不出来不是写作障碍，是"你还没有想法" |

## 常见问题

### Q: 注册时提示邮箱不被允许？
A: 确认使用的是 `@eazillion.com` 或 `@vim-technology.com` 邮箱。个人邮箱（Gmail、QQ 等）无法注册。

### Q: 上传文件大小有限制吗？
A: Supabase 免费版单文件上限 50MB，一般的 .skill 文件完全够用。

### Q: 可以更新已发布的 Skill 吗？
A: 目前需要发布新版本。后续会支持版本管理和就地更新。

### Q: 其他同事如何找到我发布的 Skill？
A: 通过 Explore 页面搜索标题、描述关键词，或按分类筛选。也可以直接分享 Skill 页面链接。
